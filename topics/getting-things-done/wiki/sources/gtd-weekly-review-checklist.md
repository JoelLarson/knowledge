This source is an official checklist for the GTD weekly review. It is one of the clearest operational references in the corpus because it reduces review into explicit steps.

## Checklist Structure

- Get Clear
- Get Current
- Get Creative

## Key Steps Captured

- collect loose inputs
- get inboxes to zero
- empty the head
- review action lists
- review past and upcoming calendar
- review waiting-for items
- review project and larger outcome lists
- review someday/maybe

## Key Takeaways

- review should use deterministic checks before any optional model summarization
- the review loop should explicitly account for waiting-for, projects, and someday/maybe states
- the system should preserve the distinction between clearing inputs, current commitments, and creative horizon scanning
