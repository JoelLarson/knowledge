GTD is not only a thinking model; it also has a setup layer that adapts the method into concrete tools and interfaces. The presence of multiple official setup guides suggests that implementation details materially affect whether GTD works in practice.

## Why It Matters

- the same GTD ontology gets expressed differently depending on the task system
- method fidelity depends on clear bucket mapping, review affordances, and low-friction capture
- users trust a system more when the structure matches the method
