Review is the maintenance loop that keeps a GTD system trustworthy. Without regular review, buckets degrade, next actions go stale, and the system becomes noise.

## GTD Function

- revisit open loops
- refresh project state
- ensure every active outcome has a next action
- identify stale, vague, or misplaced items

## Candidate Review Checks

- projects missing next actions
- stale tasks
- overloaded today lists
- long-lived unclear items
- repeatedly rejected rewrites
