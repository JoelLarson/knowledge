Someday/maybe items are meaningful possibilities that are not active commitments right now. They should be retained without pretending they are executable current work.

## Groundloop Implications

- Groundloop should be able to classify speculative or deferred items separately from active work
- review should surface likely someday/maybe items that are cluttering active lists
- this remains advisory in v1; the system should not silently relocate or rewrite core task state
