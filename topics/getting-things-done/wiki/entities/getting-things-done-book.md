*Getting Things Done* is the primary textual source for the GTD system. Multiple copies of the book are present in `raw/`, and together they provide the strongest reference base for GTD workflow assumptions.

## Why It Matters

- it defines the canonical GTD workflow
- it explains the role of projects, next actions, and regular review
- it supports the design choice to build a constrained pipeline instead of an autonomous agent
