Martin Fowler is Chief Scientist at ThoughtWorks, author of _Refactoring_ and _Patterns of Enterprise Application Architecture_, and a central figure in agile software development.

## Key Contributions

- **Refactoring** — defined and cataloged refactoring as a discipline; authored [Refactoring: Improving the Design of Existing Code](../sources/refactoring-martin-fowler.md) (1st ed. 1999, 2nd ed. 2019)
- **Patterns of Enterprise Application Architecture** — data access and architectural patterns catalog
- **UML Distilled** — practical UML reference
- **ThoughtWorks** — leadership in agile practices, continuous delivery, microservices

## Fowler's First Rule of Refactoring

> "The first step in refactoring is always the same — make sure you have a solid set of tests for that section of code."

This principle directly connects refactoring to TDD: you cannot safely refactor without tests.

## The Theatrical Players Example

Fowler's _Refactoring_ 2nd edition uses the Theatrical Players billing system as Chapter 1's worked example. This example was adapted by Emily Bache into the [Theatrical Players Refactoring Kata](../sources/fowler-theatrical-players-kata.md), a widely used practice exercise.
